Using The WiFi Ninja’s Toolkit

A practical field guide for navigating the toolkit, protecting saved credentials, and running network actions with the right amount of caution.

Version v{{ app_version }} Local toolkit build

Finding tools

Primary navigation lives in the left pane. Use Favorites for your regular shortcuts, Fortinet Tools for FortiGate and FortiAuthenticator profile pages, Network Tools for vendor-neutral diagnostics, and Administration for users, access profiles, backups, and server settings.

Fortinet-specific actions live on their FortiGate or FortiAuthenticator pages so the required saved profile context is visible before you run them.

Accounts and access

Administrators can create users and custom access profiles in Settings. A standard user only sees tools assigned through their profiles. If a page is missing from the sidebar, that is usually an access-profile choice, not a broken link.

The built-in Administrator profile is intentionally protected. Assign it only to people who should manage users, backups, server access, and high impact tools.

Profiles, secrets, and backups

Saved profiles make repeated work faster, but some contain API keys, shared secrets, passwords, SNMP credentials, or similar sensitive values. Secret fields are write-only in the UI; the toolkit does not render saved secret values back into forms.

Backup exports require encryption whenever selected data contains secrets. If a backup does not include secrets, encryption is optional. Treat any encrypted backup as sensitive because anyone with both the file and its password can restore the stored data.

Running tools safely

Read preview screens before applying changes. Export and lookup tools are generally low impact; rename, cleanup, replay, scanning, authentication, and API tools deserve more attention because they can touch infrastructure or generate traffic.

Several tools use the shared loading animation while server-side work is happening. If a target device is unreachable, a page may wait for the configured timeout before returning an error.

Packet Replay

Packet Replay can transmit raw Ethernet frames from the toolkit host. Use it only on networks where you are authorized to test. Preview the decoded fields, selected interface, repeat count, interval, VLAN fanout, and rewrite options before sending.

On macOS and Linux, raw packet permissions can vary by interface and environment. If packets appear to send but are not visible in capture, verify the capture filter first; VLAN-tagged traffic often needs a VLAN-aware filter such as vlan.

Troubleshooting

  • Use the Test button on saved FortiGate/FortiAuthenticator profiles before running profile-backed actions.
  • If a page hangs briefly, check whether the target device is offline or blocked by a firewall.
  • If the toolkit was started with sudo and later will not start normally, run the permissions repair helper.
  • If access looks wrong, check the user’s assigned profiles in Settings.
  • If a browser layout looks odd, try a hard refresh after pulling updates.
